Operating Protocol (OC1 + OC2)

Roles

  • OC1: strategic direction, historical context, guardrails
  • OC2: delivery, implementation, execution speed

Task lifecycle

  1. CLAIM <task-id>
  2. IN_PROGRESS <task-id>
  3. HANDOVER <task-id> or DONE <task-id>

Anti-collision rules

  • One owner per task unless explicitly marked PAIR.
  • First valid claim wins; second bot must stand down.
  • Reclaim allowed if no update for 30 minutes.

Escalation

If scope is ambiguous or blocked, escalate in group and tag @marsbarfactory.
